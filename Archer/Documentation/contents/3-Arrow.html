Arrow

Arrow

Introduce each attribute of an arrow object.

Name

Name of an arrow. Case sensitive. Cannot contain "white space" in the Name.
Name format:[name][.index][.type]
Both the [name] and [type] can be empty. There are some reserved [type]: "f", "u", "t", "c", "s", "i", "C#"(not case sensitive). Their meaning,
  • "f" :local file system path or short name
  • "u" :url, arguments of url will be escaped before launch
  • "t" :text, same as url. But it will set selected text as argument and launch. Useful when searching sth.
  • "c" :copy the text in arrow's Cmd and Arg then paste it.
  • "s" :simulate stroke of hardware inputs, both mouse and keyboard. Goto statement syntax.
  • "i" :internal functions of Archer
  • "C#":internal C# script, there are some special syntax for C# Args.
Such as "abc","1.2.3","!@#","index.html","python.py","fso.vbs".
If left the [type] empty, Archer will recognize it intelligently.
Different arrows can have one same [name]. Archer only treat the content before first dot as [name].
  • If the name box is focused, all the same [name]ed arrows will be launched at one time, which is useful when a project need to open a set of websites, folders and files. Archer doesn't care about the strings after the [name], so you can define a sequence like "queue.1.f" and "queue.2.f", archer will run both of them by the ASCII order.
  • If the name box is not focused, Archer will only find the first arrow which matches the whole name.
"f" type with path list. Cmd likes:
*c:\
%programfiles%\python27\python.exe
c:\python27\python.exe
You can set a path list in Cmd, each line contain a path item.
If first path doesn't exist, Archer will neither try next until find a valid path nor report an error.
Start directory is optional. The first path must start with "*". Default start directory is the path's parent directory.
Note: set the short name by your intuition, in which way you don't need to memorize it by rote and won't forget it.
Note: the reserved [type] cannot be the system registered script extension.

Stroks can contain keyboard and mouse inputs. Statement syntax:

  • Strokes should be separated by white space(s).
  • Original stroke code indicates press and then release.
  • Prefix "~" before a stroke code indicates press.
  • Prefix "!" indicates release.
  • Prefix "@" indicates the time to wait before next stroke. Such as "@10" means delay 10ms
    • .

Addinal command:

Minimize window: Min
Maximize window: Max
Make window top most: Top

Mouse key:

Move: MV(x, y)
Left button: ML
Right button: MR
Wheel: MW(d)
Middle button: MM
XButton1: M1
XButton2: M2

You can set further information of the mouse stroke like MV(100, 200) to set the position.
By default, The position is relatively to the last mouse position.
You can add "%" at the end of the statement like MV(50, 50)% (it will move mouse to the center of screen).

Set MW(-300) to set the the delta value of the wheel scrolling.

The followings are legal :

  • A
  • ~A !A (press down A and then release A, the same function as the line above)
  • ~wIn s !win (not case sensitive)
  • ~Ctrl V !Ctrl Return Escape (paste sth from clipboard then press return then press escape)
  • MV(50,50)% ML ~Ctrl A !Ctrl (mouse click center of the screen then select all)

Cmd

Preset commands. Can be url, file path, folder path or any type of text file content.
Only Cmd and Arg can be encrypted. Archer will encrypt the string via AES algorithm (the salt and the vector are constant strings, you can decompile Archer to get them).
You can set different password to different Cmd which means you needn't to use your account password.
Be carefully when you are using several passwords. If you forget the password, it will be be extremely hard to decrypt the corresponding Cmd.

Arg

Preset arguments. These arguments will be insert into the Cmd.
Arguments are separated by space. If one argument contains space, it should be surrounded by quotes.
Arg will append to the end of the Cmd by default.
For the url, file path and directory path, you can insert the Arg to specified place by using:
"<name default_value/>" tag in the Cmd. Name cannot contain white space, it is used as an hint. Such as,
Name:p.f
Cmd :D:\picture_<type sun/>_<number 00/>.jpg
In the arguments box input"moon 02", the arguments will be insert into the Cmd, the result is "D:\picture_moon_02.jpg"
Note: in C# type, argument that starts with "!" will be treated as the reference DLL path. Such as "!System.Data.dll".

Tag

Information about the arrow, tags or comments.

Hotkey

Hotkey statement. All the hotkeys will be registered globally. Because powerful always harmful, be careful when using this function. There's some simple rules about the statement:
  • Hotkey can be keyboard hotkey or mouse stroke hotkey.
  • If it's a keyboard hotkey, it should have one and only one normal key.
  • keyboard keys should be separated by white space(s).
  • If it's a mouse stroke hotkey, it should start with a "@".
  • Mouse stroke hotkey only has four directions, left, up, right, down:"<", "^", ">", "v".
  • Mouse stroke is triggered when mouse right button down.
The followings are legal :
F2
Win S
wIn s (not case sensitive)
Ctrl Shift D
Alt Ctrl Shift Win F1
@ ^ > v^ (not space sensitive)
Whitespace in the statement will not affect the result. The available key information: Key List.

Count

Record how many times user used the arrow. Useful when deleting items user don't use very often.
